# وثيقة الخطة التقنية والمواصفات الكاملة: Ebraa ERP
**المسودة الشاملة للمبرمج (Product Requirements Document - PRD)**

تجمع هذه الوثيقة كافة الرؤى الفنية، الميزات المطلوبة، وقصص المستخدم (User Stories) لبناء نظام Ebraa ERP كنسخة SaaS متكاملة، مع التركيز على كفاءة المطور المنفرد (Solo Developer) واستخدام أدوات الذكاء الاصطناعي.

---

## 1. رؤية المنتج (Product Vision)
بناء نظام ERP سحابي "نحيف وسريع" (Lean SaaS) متخصص لمحلات الإلكترونيات وقطع الغيار والبلايستيشن، يحل مشكلة تتبع الأجهزة عبر السيريال الخاص (Serial Number) أو العام للنوع، ويوفر تجربة جرد سريعة باستخدام الموبايل.

---

## 2. الهيكل التقني (Tech Stack)
يجب الالتزام بالبساطة مع القابلية للتوسع (Scalability):

- **Backend:** Laravel 11 (Strict Requirement for Inertia v3).
- **PHP Version:** PHP 8.2+ (Required).
- **Frontend:** Vue.js 3 + Inertia.js v3.0 + Vite.
- **State Management:** Pinia (لحفظ حالة السلة في POS).
- **Database:** PostgreSQL (لدعم بيانات JSONB المعقدة).
- **Real-time:** Laravel Reverb (للإشعارات اللحظية والطلبات).
- **Multi-tenancy:** Single Database مع `tenant_id` (Row Level Isolation).
- **Caching & Queues:** Redis.
- **Scanning:** WebAssembly + كاميرا المتصفح للجرد السريع.
- **Infrastructure:** Dockerized setup, AWS or DigitalOcean (Region: Frankfurt or Bahrain for Gulf).

---

## 3. الخارطة البرمجية للميزات (Feature Roadmap)

### المرحلة الأولى: النواة (MVP - Core)
- **إدارة المخازن (Serial-first Inventory):**
    - إضافة أصناف مع دعم "التتبع بالسيريال" أو "الكمية العادية".
    - طباعة باركود خاص بكل قطعة.
    - جرد المخزن بكاميرا الموبايل (Web-based scanner).
- **نقطة البيع (POS):**
    - واجهة سريعة تدعم البحث بالسيريال أو الباركود.
    - دعم طرق دفع متعددة (نقدي، فيزا، تقسيط).
    - إصدار فواتير حرارية (80mm) وفواتير A4.
- **إدارة العملاء والموردين:**
    - سجل كامل لكل عميل ومشترياته وضمان أجهزته.

### المرحلة الثانية: الصيانة والنمو
- **مديول الصيانة (Repair Center):**
    - استقبال الأجهزة (الحالة، العطل، الملحقات، الرقم السري).
    - تتبع حالة الإصلاح (انتظار، قيد العمل، تم الإصلاح، تم التسليم).
    - تقدير التكلفة وقطع الغيار المستخدمة من المخزن.
- **نظام التمويل والاشتراكات:**
    - التكامل مع بوابة دفع (Paymob/Moyasar).
    - لوحة تحكم لإدارة اشتراكات المحلات (SaaS Admin).
- **نظام الأفلييت (Affiliate):**
    - لوحة للمسوقين لتتبع عملاءهم وعمولاتهم.
- **تطبيق سطح المكتب (Desktop App):**
    - تغليف النظام باستخدام NativePHP أو Tauri لتحسين تجربة الطباعة والسرعة.

### المرحلة الثالثة: الأتمتة والتوسع
- **الواتساب (WhatsApp API):** إرسال الفواتير وحالات الصيانة آلياً للعملاء.
- **التجارة الإلكترونية (Mini-Ecommerce):** تحويل المخزن لصفحة عرض أونلاين بضغطة زر.
- **الربط الضريبي:** ETA (مصر) و ZATCA (السعودية).

## 4. قصص المستخدم (User Stories)

### أ- صاحب المحل (Owner)
- "بصفتي صاحب محل، أريد رؤية تقرير أرباحي اليومي لحظياً من موبايلي لأتابع سير العمل وأنا خارج المحل."
- "بصفتي صاحب محل، أريد نظاماً يمنع بيع أي قطعة لها سيريال إلا بعد تسجيله، لمنع السرقة والتلاعب."

### ب- موظف المبيعات (Salesman)
- "بصفتي بائع، أريد واجهة POS سريعة لا تتطلب إعادة تحميل الصفحة لإتمام عملية البيع في أقل من 10 ثوانٍ."
- "أريد معرفة ما إذا كان الجهاز تحت الضمان بمجرد مسح السيريال الخاص به."

### ج- فني الصيانة (Technician)
- "بصفتي فني، أريد تسجيل قطع الغيار التي استهلكتها في إصلاح جهاز معين لتخصم من المخزن وتضاف لتكلفة الصيانة آلياً."
- "أريد رفع صور لحالة الجهاز قبل الإصلاح لحماية المحل من ادعاءات العملاء."

## 5. المواصفات الفنية المتقدمة (Developer Guidelines)

### عزل البيانات (Multi-tenancy Strategy)
يجب بناء `TenantScope` يطبق تلقائياً على كافة الـ Models لضمان عدم تسريب بيانات محل لمحل آخر:
```php
// Example Logic
static::addGlobalScope('tenant', function (Builder $builder) {
    $builder->where('tenant_id', auth()->user()->tenant_id);
});
```

### إدارة الأجهزة بالسيريال (The Serial Table)
بدلاً من تخزين السيريال كنص في جدول المبيعات، يفضل جدول منفصل `device_items`:
- `id`, `product_id`, `serial_number`, `status` (in_stock, sold, in_repair), `tenant_id`.

### الجرد الذكي (Mobile Scanning)
استخدام مكتبة `Html5-QRCode` وتطوير واجهة "جرد سريعة" تقارن الممسوح برمجياً مع المخزن وتعطي تنبيهات صوتية (Beep) عند الخطأ أو النجاح.

## 6. خطة التنفيذ (Implementation Plan)

1.  **الأسبوع 1-2:** بناء الـ Boilerplate، إعداد Multi-tenancy، ونظام الصلاحيات (Spatie Permissions).
2.  **الأسبوع 3-5:** تطوير مديول المخازن والمنتجات (دعم السيريال).
3.  **الأسبوع 6-8:** بناء واجهة الـ POS وربط الطابعات الحرارية.
4.  **الأسبوع 9-11:** تطوير مديول الصيانة وتدفق العمل (Workflow).
5.  **الأسبوع 12-14:** نظام الاشتراكات، بوابات الدفع، ولوحة تحكم الأدمن الرئيسي.
6.  **الأسبوع 15-16:** الاختبارات النهائية (UAT) والإطلاق التجريبي (Beta).

## 7. تعليمات للمطور (Solo Developer Tips)
- **الذكاء الاصطناعي:** استخدم Cursor/Claude لكتابة الـ CRUDs والـ Unit Tests لتوفير 60% من وقت الكود المتكرر.
- **التوثيق:** وثق الـ API باستخدام `Scribe` أو `Swagger` لتسهيل بناء تطبيق موبايلي مستقبلاً.
- **الأداء:** استخدم `Spatie Query Builder` لتسهيل عمليات الفلترة المعقدة في التقارير.
- **الأمان:** تأكد من عمل Backup يومي لـ PostgreSQL خارج السيرفر (S3 bucket).
