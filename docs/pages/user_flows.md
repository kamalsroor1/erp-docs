# 🚀 رحلات المستخدم (User Flows) - Ebraa ERP
**المنطق التفاعلي لمسارات العمل الأساسية v1.0**

---

## 1. المسار السعيد: عملية بيع بالسيريال (Core Happy Path)
*الهدف: إتمام البيع في أقل من 10 ثوانٍ.*

```mermaid
graph TD
    A[بداية: شاشة POS] --> B{مسح سيريال/باركود}
    B -- "موجود" --> C[إضافة لحظية للسلة - Optimistic UI]
    B -- "غير موجود" --> D[تنبيه خطأ بصرية + صوت تنبيه]
    C --> E{هل توجد أصناف أخرى؟}
    E -- "نعم" --> B
    E -- "لا" --> F[ضغط F4: إتمام الدفع]
    F --> G{اختيار نوع الدفع}
    G -- "نقدي" --> H[إدخال المبلغ المستلم]
    G -- "آجل" --> I[اختيار العميل من سجل الديون]
    H --> J[حفظ الفاتورة]
    I --> J
    J --> K[طباعة الفاتورة + إشعار تليجرام]
    K --> L[تصفير الشاشة والعودة لحقل البحث]
```

---

## 2. المسار الثانوي: فتح تذكرة صيانة (Repair Intake Path)
*الهدف: توثيق حالة الجهاز وتأمين حق التاجر.*

```mermaid
graph TD
    A[بداية: وحدة الصيانة] --> B[زر: تذكرة جديدة]
    B --> C{بحث عن عميل بالموبايل}
    C -- "موجود" --> D[اختيار العميل]
    C -- "جديد" --> E[إدخال الاسم والموبايل]
    D --> F[إدخال بيانات الجهاز: ماركة، موديل، سيريال]
    E --> F
    F --> G[وصف العطل + فحص ظاهري]
    G --> H[تحديد التكلفة التقديرية وموعد التسليم]
    H --> I[حفظ التذكرة]
    I --> J[طباعة ملصق الباركود للجهاز]
    J --> K[إرسال رابط تتبع حي للعميل عبر تليجرام]
```

---
*ملاحظة تقنية: كافة الانتقالات بين الخطوات تتم عبر Inertia.js دون إعادة تحميل الصفحة لضمان تجربة Zero-Lag.*
