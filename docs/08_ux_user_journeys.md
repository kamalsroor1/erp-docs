# 08 رحلات المستخدم (User Journeys)
**مشروع Ebraa ERP - تدفقات الاستخدام الفعلية**
*النسخة v1.0 - تجربة مستخدم موجهة للكفاءة*

---

## 🚀 رحلة 1: البائع (إتمام عملية بيع IMEI في 15 ثانية)
**الهدف:** مسح سيريال الجهاز، إتمام البيع، والحصول على الفاتورة دون استخدام الماوس.

| الخطوة | الإجراء (Action) | تجربة النظام (System Reaction) | المشاعر (Emotions) | الحل التصميمي |
|---|---|---|---|---|
| **1. البدء** | يضغط `F1` من أي مكان | فتح شاشة POS فوراً مع Focus على حقل الـ IMEI | ⚡ سرعة | اختصار لوحة مفاتيح عالمي (NativePHP) |
| **2. المسح** | يمرر ماسح الباركود على علبة الجهاز | قراءة السيريال، التحقق اللحظي من توفره في المخزن | 🛡️ اطمئنان | Real-time Validation (Livewire/Vue) |
| **3. الإضافة** | الجهاز يُضاف تلقائياً للسلة | عرض اسم المنتج، السعر، وحالة الضمان فوراً | 😃 رضا | تحديث واجهة بدون Reload |
| **4. الدفع** | يضغط `Enter` ثم يكتب المبلغ | حساب المتبقي (Change) وتفعيل زر الطباعة | ✅ دقة | لوحة دفع منبثقة سريعة (Modal) |
| **5. النهاية** | يضغط `Ctrl+P` | طباعة صامتة فورية (Silent Print) وإفراغ السلة | 🏆 إنجاز | NativePHP Local Printing Integration |

## 🤝 رحلة 2: المسوق (تتبع الأرباح وسحب العمولة)
**الهدف:** متابعة نجاح الإحالات والوصول للرصيد المتاح للسحب بسهولة.

| الخطوة | الإجراء (Action) | تجربة النظام (System Reaction) | المشاعر (Emotions) | الحل التصميمي |
|---|---|---|---|---|
| **1. الدخول** | تسجيل الدخول لبوابة `/affiliate` | لوحة تحكم بسيطة تعرض "رصيد الأرباح" بخط كبير | 💰 تحفيز | بطاقات KPI بارزة (Emerald Design) |
| **2. التتبع** | الضغط على "سجل الإحالات" | عرض قائمة المحلات (نشط/تجريبي) مع قيمة العمولة لكل واحد | 📈 وضوح | جدول بيانات مرتب بوضوح (Status Badges) |
| **3. المراقبة** | متابعة وصول الرصيد لـ 500 ج.م | تفعيل زر "طلب سحب" (Request Payout) تلقائياً | 🤩 حماس | Conditional UI Rendering (أزرار مفعلة برمجياً) |
| **4. الطلب** | اختيار وسيلة الدفع (InstaPay) | إرسال طلب للـ SaaS Admin وتلقي إشعار تأكيد | 🕊️ ثقة | نظام إشعارات فوري داخل البوابة |

## 📱 رحلة 3: صاحب المحل (الجرد السريع بالموبايل)
**الهدف:** التأكد من مطابقة الأجهزة الموجودة فعلياً لما هو مسجل في النظام.

| الخطوة | الإجراء (Action) | تجربة النظام (System Reaction) | المشاعر (Emotions) | الحل التصميمي |
|---|---|---|---|---|
| **1. التفعيل** | يفتح "الجلسة" من الكمبيوتر | توليد QR Code لبدء الجلسة من الموبايل | 🏗️ تنظيم | الربط بين الـ Desktop والـ Mobile Web |
| **2. المسح** | يفتح الكاميرا من الموبايل ويمسح سيريالات الأرفف | اهتزاز خفيف (Vibration) مع كل مسح ناجح | 🛠️ إنتاجية | Mobile Camera API + Haptic Feedback |
| **3. التنبيه** | مسح سيريال مسجل كـ "مباع" أو غير موجود | صوت تنبيه (Beep) وتحذير بلون أحمر | ⚠️ انتباه | Audible Alerts + Visual Warnings |
| **4. المقارنة** | يضغط "إنهاء الجرد" | توليد تقرير فوراً: (موجود / مفقود / زيادة) | 🧐 تحكم | خوارزمية مقارنة لحظية (Difference Report) |

---

## 💡 نقاط القوة في هذه الرحلات (UX Core Values)
- **ردود الفعل اللحظية (Immediate Feedback):** النظام دائماً يخبر المستخدم ماذا يحدث (صوت، اهتزاز، إشعار).
- **تقليل التشتت:** كل رحلة تركز على هدف واحد فقط في الصفحة.
- **تكامل البيئات:** التنقل السلس بين Desktop و Mobile و Web بوابة الأفلييت.
