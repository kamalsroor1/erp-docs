# 01 خارطة طريق المشروع (Project Roadmap)
**مشروع Ebraa ERP - المرحلة الأولى (MVP Phase 1)**
*النسخة v1.1 - مراجعة ومحسّنة*

---

## 1. ملخص المرحلة الأولى (MVP Overview)
الهدف من المرحلة الأولى هو بناء النواة الصلبة للنظام التي تسمح لمحلات الإلكترونيات بـ:
- إدارة مخزونها بدقة عبر الأرقام التسلسلية (Serial Numbers).
- إتمام عمليات البيع بسرعة من خلال واجهة POS احترافية.
- مراقبة الأرباح لحظياً من لوحة تحكم صاحب المحل.
- ضمان عزل بيانات كل عميل (Multi-tenancy) بشكل محكم.

**الفترة الزمنية:** 16 أسبوع
**المنهجية:** Agile Scrum (دورات أسبوعية × 2)
**المطور:** Solo Senior Developer + AI Tools

---

## 2. نطاق العمل (Project Scope)

### ✅ داخل النطاق (In Scope - MVP Phase 1):
| المجال | الميزات المشمولة |
|---|---|
| **البنية التحتية** | إعداد السيرفر، Docker، Multi-tenancy، Auth + Roles |
| **إدارة المخازن** | منتجات بسيريال وبكمية، جرد بالموبايل، استيراد Excel |
| **نقطة البيع (POS)** | واجهة بيع سريعة، دعم الباركود، طرق دفع متعددة، طباعة |
| **الفوترة** | فواتير A4 + حرارية، مرتجعات، خصومات |
| **الموردون والمشتريات** | أوامر الشراء، استلام البضاعة، ديون الموردين |
| **التقارير الأساسية** | الأرباح اليومية، أكثر المنتجات مبيعاً، حركة الخزينة |
| **لوحة الـ SaaS Admin** | إدارة الاشتراكات، تفعيل المحلات، بوابة الدفع |

### ❌ خارج النطاق (Out of Scope - مرحلة لاحقة):
- تطبيق سطح المكتب الأصلي (NativePHP/Tauri) ← المرحلة الثانية
- مديول الصيانة المتقدم (Repair Center) ← المرحلة الثانية
- الربط مع الفاتورة الإلكترونية (ETA مصر / ZATCA السعودية) ← المرحلة الثالثة
- تطبيق موبايل أصلي (Native iOS/Android) ← المرحلة الثالثة
- ربط WhatsApp API ← المرحلة الثانية
- نظام الأفلييت ← المرحلة الثانية
- التجارة الإلكترونية (Mini-Ecommerce) ← المرحلة الثالثة

## 3. الجدول الزمني التفصيلي (16 أسبوع)

### 🏗️ المرحلة 1: التأسيس والبنية التحتية (الأسبوع 1 - 3)
**الهدف:** بيئة تطوير مستقرة وجاهزة لبناء الميزات.

| الأسبوع | المهام الرئيسية |
|---|---|
| **1** | إعداد Laravel 11، Docker، CI/CD pipeline أساسي |
| **2** | تطبيق Multi-tenancy + نظام الصلاحيات (Roles & Permissions) |
| **3** | Authentication كامل (تسجيل، دخول، استعادة كلمة المرور) |

**✅ شروط اكتمال الميلستون 1:**
- بيئة Docker تعمل محلياً وعلى السيرفر.
- لا يمكن لمستخدم من Tenant A رؤية بيانات Tenant B (اختبار موثق).
- نظام الأدوار يمنع الوصول غير المصرح به.

---

### 📦 المرحلة 2: إدارة المنتجات والمخازن (الأسبوع 4 - 7)
**الهدف:** تسجيل كل قطعة في المخزن بسيريالها الفريد.

| الأسبوع | المهام الرئيسية |
|---|---|
| **4** | CRUD كامل للمنتجات (تصنيفات، ماركات، وحدات قياس) |
| **5** | بناء جدول `device_items` ومنطق التتبع بالسيريال |
| **6** | واجهة إضافة مخزون مع ماسح الباركود بالكاميرا |
| **7** | استيراد منتجات من Excel + طباعة باركودات |

**✅ شروط اكتمال الميلستون 2:**
- إضافة منتج serialized وربطه بـ 10 سيريالات مختلفة بنجاح.
- النظام يرفض إدخال سيريال مكرر لنفس الـ Tenant.
- الجرد بالكاميرا يعمل على متصفح الموبايل (Chrome/Safari).

---

### 🛒 المرحلة 3: نقطة البيع والفوترة (الأسبوع 8 - 11)
**الهدف:** واجهة POS احترافية لإتمام عملية بيع كاملة في أقل من 10 ثوانٍ.

| الأسبوع | المهام الرئيسية |
|---|---|
| **8** | بناء واجهة POS بـ Vue.js 3 (SPA) |
| **9** | منطق البحث بالسيريال، الخصومات، وطرق الدفع المتعددة |
| **10** | طباعة الفاتورة (Thermal 80mm + A4) وربط الطابعة |
| **11** | نظام المرتجعات + إدارة الديون الآجلة |

**✅ شروط اكتمال الميلستون 3:**
- إتمام عملية بيع كاملة (من البحث للطباعة) في أقل من 10 ثوانٍ.
- الفاتورة المطبوعة تحتوي على السيريال + تاريخ الضمان بدقة.
- حالة السيريال تتحول لـ "Sold" فوراً بعد البيع.

---

### 📊 المرحلة 4: التقارير والإدارة المالية (الأسبوع 12 - 14)
**الهدف:** تمكين صاحب المحل من معرفة وضعه المالي بدقة لحظياً.

| الأسبوع | المهام الرئيسية |
|---|---|
| **12** | مديول الخزينة (وارد/منصرف نقدي) |
| **13** | تقارير المبيعات، الأرباح، والمنتجات الأكثر مبيعاً |
| **14** | لوحة تحكم صاحب المحل (Dashboard) + تقارير Excel |

**✅ شروط اكتمال الميلستون 4:**
- تقرير الأرباح يطابق الحسابات اليدوية بدقة 100%.
- لوحة التحكم تعرض مؤشرات اليوم لحظياً.

---

### 🚀 المرحلة 5: الـ SaaS، الاختبار، والإطلاق (الأسبوع 15 - 16)
**الهدف:** تحويل النظام لمنتج تجاري قابل للاشتراك وإطلاقه للعملاء الأوائل.

| الأسبوع | المهام الرئيسية |
|---|---|
| **15** | لوحة Super Admin + ربط Paymob للاشتراكات |
| **16** | اختبارات الأداء، إصلاح الثغرات، إعداد Production Server |

**✅ شروط اكتمال الميلستون 5 (إطلاق Beta):**
- اكتمال Onboarding لـ 3 عملاء تجريبيين بدون تدخل بشري.
- تجديد اشتراك واحد على الأقل يتم بشكل آلي عبر Paymob.
