## 1. التعارضات المكتشفة في الملفات السابقة

بعد مراجعة الملفات، وُجدت التعارضات التالية مع مبدأ الـ Multi-tenancy:

| # | مكان التعارض | المشكلة | الحل |
|---|---|---|---|
| 1 | ملف 04 - قسم DB Schema | جدول `tenants` مدرج مع جداول الـ Tenant وهو خطأ هيكلي | **نقله لقاعدة البيانات المركزية** |
| 2 | ملف 04 - الـ BaseModel | الكود يقول "كل Model يرث من BaseModel بدون استثناء" لكن الاستثناء مذكور بعدها مباشرة — تناقض | **تعريف نموذجين: `CentralModel` و `TenantModel`** |
| 3 | ملف 04 - قسم Multi-tenancy | الـ Global Scope يُطبَّق إذا كان `auth()->check()` فقط — هذا يعني Super Admin يرى بيانات أول Tenant يجدها | **ربط Super Admin بـ DB مختلفة وليس بـ tenant_id** |
| 4 | ملف 03 - الـ Epics | لا يوجد Epic للـ Affiliate System رغم أنه مذكور في دراسة الجدوى كمصدر دخل | **أُضيف Epic 7: Affiliate System** |
| 5 | ملف 01 - الـ Scope | الأفلييت مذكور كـ Out of Scope بدون سبب | **إعادة تصنيفه كـ In Scope (MVP Phase 1 نهاية)** |

## 2. المعمارية المُصحَّحة: قاعدتا بيانات (Two-Database Architecture)

┌─────────────────────────────────────────────┐
│           DATABASE 1: Central DB            │
│          (قاعدة بيانات مركزية)              │
│                                             │
│  إدارة الـ SaaS والاشتراكات والأفلييت        │
│  ┌─────────┐  ┌───────┐  ┌──────────────┐  │
│  │  plans  │  │tenants│  │  domains     │  │
│  └─────────┘  └───────┘  └──────────────┘  │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│           DATABASE 2: Tenant DBs            │
│    (قواعد بيانات منفصلة لكل محل)            │
│                                             │
│  يتم إنشاء قاعدة بيانات جديدة لكل Tenant     │
│  ┌───────┐ ┌──────────┐ ┌──────────────┐   │
│  │ users │ │ products │ │ device_items │   │
│  └───────┘ └──────────┘ └──────────────┘   │
└─────────────────────────────────────────────┘

### لماذا قاعدتان وليس واحدة؟
- **السبب 1 - الأمان:** بيانات الـ SaaS Admin (العقود، الاشتراكات، العمولات) أكثر حساسية ويجب أن تكون في DB منفصلة مع صلاحيات مختلفة.
- **السبب 2 - الأداء:** تقارير الـ Super Admin (كم محل اشترك؟ كم عمولة دفعنا؟) لا تحتاج للبحث في ملايين سجلات المخازن.
- **السبب 3 - التوسع:** في المستقبل يمكن رفع الـ Tenant DB إلى سيرفر منفصل بدون التأثير على مركزية SaaS.

## 3. Schema قاعدة البيانات المركزية (Central DB)

```sql
-- ============================================
-- DATABASE 1: Central / SaaS Database
-- ============================================

-- الباقات المتاحة للاشتراك
CREATE TABLE plans (
    id              BIGSERIAL PRIMARY KEY,
    name            VARCHAR(100) NOT NULL,          -- Starter, Professional, Enterprise
    slug            VARCHAR(100) UNIQUE NOT NULL,
    price_egp       DECIMAL(10,2) NOT NULL,          -- السعر بالجنيه
    price_sar       DECIMAL(10,2),                   -- السعر بالريال (للخليج)
    max_users       INT DEFAULT 3,
    max_branches    INT DEFAULT 1,
    features        JSONB,                           -- قائمة الميزات بصيغة JSON
    is_active       BOOLEAN DEFAULT true,
    created_at      TIMESTAMP DEFAULT NOW()
);

-- المحلات المشتركة (Tenants)
CREATE TABLE tenants (
    id              BIGSERIAL PRIMARY KEY,
    name            VARCHAR(255) NOT NULL,           -- اسم المحل
    slug            VARCHAR(100) UNIQUE NOT NULL,    -- serialtrack.app/{slug}
    plan_id         BIGINT REFERENCES plans(id),
    status          VARCHAR(20) DEFAULT 'trial',     -- trial, active, suspended, cancelled
    trial_ends_at   TIMESTAMP,
    country         VARCHAR(10) DEFAULT 'EG',        -- EG, SA, AE
    currency        VARCHAR(5) DEFAULT 'EGP',
    affiliate_id    BIGINT REFERENCES affiliates(id) NULL, -- من سجّل هذا المحل
    created_at      TIMESTAMP DEFAULT NOW(),
    updated_at      TIMESTAMP DEFAULT NOW()
);

-- الاشتراكات (سجل كل الفواتير السابقة)
CREATE TABLE subscriptions (
    id              BIGSERIAL PRIMARY KEY,
    tenant_id       BIGINT REFERENCES tenants(id),
    plan_id         BIGINT REFERENCES plans(id),
    amount          DECIMAL(10,2) NOT NULL,
    currency        VARCHAR(5) DEFAULT 'EGP',
    status          VARCHAR(20),                     -- paid, failed, refunded
    payment_gateway VARCHAR(50),                     -- paymob, moyasar
    gateway_ref     VARCHAR(255),                    -- رقم مرجع بوابة الدفع
    period_start    DATE NOT NULL,
    period_end      DATE NOT NULL,
    paid_at         TIMESTAMP,
    created_at      TIMESTAMP DEFAULT NOW()
);

-- مسؤولو الـ SaaS (Super Admins)
CREATE TABLE saas_admins (
    id              BIGSERIAL PRIMARY KEY,
    name            VARCHAR(255) NOT NULL,
    email           VARCHAR(255) UNIQUE NOT NULL,
    password        VARCHAR(255) NOT NULL,
    role            VARCHAR(50) DEFAULT 'admin',     -- super_admin, support, finance
    is_active       BOOLEAN DEFAULT true,
    last_login_at   TIMESTAMP,
    created_at      TIMESTAMP DEFAULT NOW()
);

-- ============================================
-- نظام الأفلييت (Affiliate System)
-- ============================================

-- المسوقون (Affiliates)
CREATE TABLE affiliates (
    id              BIGSERIAL PRIMARY KEY,
    name            VARCHAR(255) NOT NULL,
    email           VARCHAR(255) UNIQUE NOT NULL,
    password        VARCHAR(255) NOT NULL,
    phone           VARCHAR(20),
    referral_code   VARCHAR(20) UNIQUE NOT NULL,     -- الكود الفريد مثل: KAMAL2025
    referral_link   VARCHAR(500),                    -- الرابط الكامل (يُولَّد آلياً)
    commission_rate DECIMAL(5,2) DEFAULT 20.00,      -- نسبة العمولة (افتراضي 20%)
    total_earned    DECIMAL(12,2) DEFAULT 0,         -- إجمالي ما كسبه
    total_paid      DECIMAL(12,2) DEFAULT 0,         -- إجمالي ما دُفع له
    balance         DECIMAL(12,2) DEFAULT 0,         -- الرصيد المعلق (= earned - paid)
    status          VARCHAR(20) DEFAULT 'active',    -- active, suspended
    payout_method   VARCHAR(50),                     -- instapay, bank, vodafone_cash
    payout_details  JSONB,                           -- بيانات الدفع
    is_active       BOOLEAN DEFAULT true,
    created_at      TIMESTAMP DEFAULT NOW()
);

-- سجل التحويلات (كل مرة يشترك محل من رابط مسوق)
CREATE TABLE affiliate_conversions (
    id              BIGSERIAL PRIMARY KEY,
    affiliate_id    BIGINT REFERENCES affiliates(id),
    tenant_id       BIGINT REFERENCES tenants(id),   -- المحل الذي اشترك
    subscription_id BIGINT REFERENCES subscriptions(id), -- الفاتورة المحددة (دعم العمولات المتكررة)
    amount          DECIMAL(10,2) NOT NULL,           -- قيمة الاشتراك
    commission      DECIMAL(10,2) NOT NULL,           -- العمولة المحسوبة
    status          VARCHAR(20) DEFAULT 'pending',    -- pending, approved, paid
    approved_at     TIMESTAMP,
    paid_at         TIMESTAMP,
    created_at      TIMESTAMP DEFAULT NOW()
);

-- تذاكر الدعم الفني (Support Tickets)
CREATE TABLE support_tickets (
    id              BIGSERIAL PRIMARY KEY,
    tenant_id       BIGINT REFERENCES tenants(id),
    subject         VARCHAR(255) NOT NULL,
    description     TEXT NOT NULL,
    status          VARCHAR(20) DEFAULT 'open',      -- open, pending, closed
    priority        VARCHAR(20) DEFAULT 'medium',    -- low, medium, high, critical
    created_at      TIMESTAMP DEFAULT NOW()
);
```

## 4. Schema قاعدة بيانات المحلات (Tenant DB)

```sql
-- ============================================
-- DATABASE 2: Tenant Database (Row Isolated)
-- ============================================

-- الفروع (داخل قاعدة بيانات المحل)
CREATE TABLE branches (
    id              BIGSERIAL PRIMARY KEY,
    name            VARCHAR(255) NOT NULL,
    address         TEXT,
    phone           VARCHAR(20),
    settings        JSONB,                           -- إعدادات خاصة بالفرع (اللوجو، ترويسة الفاتورة)
    created_at      TIMESTAMP DEFAULT NOW()
);

-- المخازن (مربوطة بالفروع)
CREATE TABLE warehouses (
    id              BIGSERIAL PRIMARY KEY,
    branch_id       BIGINT REFERENCES branches(id),
    name            VARCHAR(255) NOT NULL,
    is_default      BOOLEAN DEFAULT false,
    created_at      TIMESTAMP DEFAULT NOW()
);

-- مستخدمو المحلات (كل مستخدم مربوط بـ tenant_id)
CREATE TABLE users (
    id              BIGSERIAL PRIMARY KEY,
    name            VARCHAR(255) NOT NULL,
    email           VARCHAR(255) UNIQUE NOT NULL,
    password        VARCHAR(255) NOT NULL,
    role            VARCHAR(50) NOT NULL,
    is_active       BOOLEAN DEFAULT true,
    telegram_chat_id VARCHAR(100) NULL
);

-- ربط المستخدمين بالفروع (Many-to-Many)
CREATE TABLE user_branches (
    user_id         BIGINT REFERENCES users(id),
    branch_id       BIGINT REFERENCES branches(id),
    PRIMARY KEY (user_id, branch_id)
);

-- التصنيفات والماركات
CREATE TABLE categories (id, name, parent_id);
CREATE TABLE brands (id, name, country_of_origin);

-- المنتجات
CREATE TABLE products (
    id              BIGSERIAL PRIMARY KEY,
    category_id     BIGINT REFERENCES categories(id),
    brand_id        BIGINT REFERENCES brands(id),
    name            VARCHAR(255) NOT NULL,
    sku             VARCHAR(100) UNIQUE,             -- باركود 1
    barcode_2       VARCHAR(100) UNIQUE,             -- باركود 2
    type            VARCHAR(20),                     -- serial, bulk, service
    average_cost    DECIMAL(15,2) DEFAULT 0,
    sell_price      DECIMAL(15,2),
    min_sell_price  DECIMAL(15,2),                   -- أقل سعر بيع مسموح
    status          VARCHAR(20) DEFAULT 'active',    -- active, inactive
    reorder_point   INT DEFAULT 5,
    warranty_months INT DEFAULT 0,
    tax_rate        DECIMAL(5,2) DEFAULT 14.00,      -- VAT 14%
    deleted_at      TIMESTAMP
);

-- أرصدة المخازن (الجدول الوسيط للأرصدة)
CREATE TABLE inventory_stocks (
    item_id         BIGINT NOT NULL,                 -- مراجع لـ products.id
    warehouse_id    BIGINT REFERENCES warehouses(id),
    quantity        DECIMAL(15,3) DEFAULT 0,         -- الرصيد الحالي في هذا المخزن
    PRIMARY KEY (item_id, warehouse_id)
);

-- القطع الفردية (السيريال - مربوطة بمخزن محدد)
CREATE TABLE device_items (
    id              BIGSERIAL PRIMARY KEY,
    product_id      BIGINT REFERENCES products(id),
    warehouse_id    BIGINT REFERENCES warehouses(id),
    serial_number   VARCHAR(255) NOT NULL,
    status          VARCHAR(20) DEFAULT 'available', -- available, sold, in_transit, in_repair, reserved
    purchase_price  DECIMAL(15,2),
    UNIQUE (serial_number) -- Unique inside this Tenant DB only
);

-- التحويلات المخزنية (Stock Transfers)
CREATE TABLE stock_transfers (
    id              BIGSERIAL PRIMARY KEY,
    from_warehouse_id BIGINT REFERENCES warehouses(id),
    to_warehouse_id   BIGINT REFERENCES warehouses(id),
    status          VARCHAR(20) DEFAULT 'pending',   -- pending, shipped, received
    notes           TEXT,
    created_by      BIGINT REFERENCES users(id),
    shipped_at      TIMESTAMP,
    received_at     TIMESTAMP,
    created_at      TIMESTAMP DEFAULT NOW()
);
CREATE TABLE stock_transfer_lines (
    id              BIGSERIAL PRIMARY KEY,
    transfer_id     BIGINT REFERENCES stock_transfers(id),
    product_id      BIGINT,
    quantity        DECIMAL(15,3) NOT NULL,
    device_item_id  BIGINT NULL
);

-- الجرد والتسويات (Inventory Adjustments)
CREATE TABLE inventory_adjustments (
    id              BIGSERIAL PRIMARY KEY,
    warehouse_id    BIGINT REFERENCES warehouses(id),
    reason          VARCHAR(255),                    -- جرد دوري، تالف، هالك
    type            VARCHAR(10),                     -- POSITIVE (زيادة), NEGATIVE (نقص)
    created_by      BIGINT REFERENCES users(id),
    created_at      TIMESTAMP DEFAULT NOW()
);
CREATE TABLE inventory_adjustment_lines (
    id              BIGSERIAL PRIMARY KEY,
    adjustment_id   BIGINT REFERENCES inventory_adjustments(id),
    product_id      BIGINT,
    quantity        DECIMAL(15,3),
    device_item_id  BIGINT NULL
);

-- العملاء والموردون
CREATE TABLE customers (id, name, phone, total_debt DECIMAL(15,2) DEFAULT 0, branch_id BIGINT REFERENCES branches(id));
CREATE TABLE suppliers (id, name, phone, balance DECIMAL(15,2) DEFAULT 0);

-- المبيعات (مربوطة بفرع ومخزن)
CREATE TABLE sales (
    id              BIGSERIAL PRIMARY KEY,
    branch_id       BIGINT REFERENCES branches(id),
    warehouse_id    BIGINT REFERENCES warehouses(id),
    customer_id     BIGINT REFERENCES customers(id),
    user_id         BIGINT REFERENCES users(id),
    total           DECIMAL(15,2),
    discount        DECIMAL(15,2) DEFAULT 0,
    tax             DECIMAL(15,2) DEFAULT 0,
    status          VARCHAR(20) DEFAULT 'completed',
    invoice_number  VARCHAR(50),
    payment_type    VARCHAR(20),                     -- cash, network, credit, partial
    paid_amount     DECIMAL(15,2) DEFAULT 0,
    remaining       DECIMAL(15,2) DEFAULT 0,
    created_at      TIMESTAMP DEFAULT NOW()
);

-- سجل الدفعات (Payment Transactions)
CREATE TABLE payment_transactions (
    id              BIGSERIAL PRIMARY KEY,
    sale_id         BIGINT REFERENCES sales(id),
    amount          DECIMAL(15,2) NOT NULL,
    payment_method  VARCHAR(20),                     -- cash, network, installment
    reference_num   VARCHAR(100),
    paid_at         TIMESTAMP DEFAULT NOW()
);
CREATE TABLE sale_items (
    id, sale_id, product_id,
    device_item_id  BIGINT NULL,
    quantity        DECIMAL(15,3) DEFAULT 1,
    unit_price      DECIMAL(15,2),
    cost_price      DECIMAL(15,2) -- التكلفة وقت البيع (Moving Average)
);

-- أوامر الشراء
CREATE TABLE purchase_orders (
    id, branch_id BIGINT REFERENCES branches(id),
    warehouse_id BIGINT REFERENCES warehouses(id),
    supplier_id, total, status, date
);
CREATE TABLE purchase_items (
    id, purchase_order_id, product_id, quantity, unit_cost
);

-- عروض الأسعار (Quotations)
CREATE TABLE quotations (
    id              BIGSERIAL PRIMARY KEY,
    branch_id       BIGINT REFERENCES branches(id),
    customer_id     BIGINT,
    total           DECIMAL(15,2),
    valid_until     DATE,
    status          VARCHAR(20) DEFAULT 'pending',   -- pending, converted, expired
    created_at      TIMESTAMP DEFAULT NOW()
);

-- المرتجعات (Returns)
CREATE TABLE returns (
    id              BIGSERIAL PRIMARY KEY,
    sale_id         BIGINT REFERENCES sales(id) NULL,
    purchase_id     BIGINT REFERENCES purchase_orders(id) NULL,
    type            VARCHAR(10),                     -- SALE, PURCHASE
    total           DECIMAL(15,2),
    created_at      TIMESTAMP DEFAULT NOW()
);

-- الخزائن (لكل فرع خزينة)
CREATE TABLE treasuries (
    id              BIGSERIAL PRIMARY KEY,
    branch_id       BIGINT REFERENCES branches(id),
    name            VARCHAR(255) NOT NULL,
    balance         DECIMAL(15,2) DEFAULT 0
);

-- المعاملات المالية (مربوطة بفرع وخزينة)
CREATE TABLE cash_transactions (
    id              BIGSERIAL PRIMARY KEY,
    branch_id       BIGINT REFERENCES branches(id),
    treasury_id     BIGINT REFERENCES treasuries(id),
    type            VARCHAR(10),                     -- IN, OUT
    amount          DECIMAL(15,2),
    reference_id    BIGINT,                          -- id الفاتورة أو السند
    reference_type  VARCHAR(50),                     -- sale, purchase, expense, return, transfer
    created_at      TIMESTAMP DEFAULT NOW()
);

-- المصروفات (Expenses)
CREATE TABLE expenses (
    id              BIGSERIAL PRIMARY KEY,
    branch_id       BIGINT REFERENCES branches(id),
    treasury_id     BIGINT REFERENCES treasuries(id),
    category        VARCHAR(100),                    -- إيجار، كهرباء، مرتبات، صيانة
    amount          DECIMAL(15,2) NOT NULL,
    description     TEXT,
    date            DATE NOT NULL,
    created_by      BIGINT REFERENCES users(id),
    created_at      TIMESTAMP DEFAULT NOW()
);

-- الأقساط (Installments)
CREATE TABLE installments (

    due_date        DATE NOT NULL,

CREATE TABLE cheques (

    status          VARCHAR(20),                     -- received, diagnosing, repairing, ready, delivered

    delivered_at    TIMESTAMP,

CREATE TABLE repair_parts (                          -- قطع الغيار المستخدمة في الصيانة

CREATE TABLE print_templates (
```

```php
// app/Http/Middleware/TenantMiddleware.php
class TenantMiddleware
{
    public function handle(Request $request, Closure $next)
    {
        $subdomain = explode('.', $request->getHost())[0]; // أو من JWT
        $tenant = Tenant::on('central')->where('slug', $subdomain)->firstOrFail();

        if ($tenant->status !== 'active') {
            return response()->json(['error' => 'Subscription expired'], 403);
        }

        app()->instance('current_tenant_id', $tenant->id);
        app()->instance('current_tenant', $tenant);

        return $next($request);
    }
}
```

## 6. نظام الأفلييت (Affiliate System) — التصميم الكامل

### 6.1 كيف يعمل النظام؟

```
[1] المسوق يُسجَّل في النظام (من Super Admin أو Self-signup)
        ↓
[2] النظام يولد له رابط فريد:
    https://serialtrack.app/register?ref=KAMAL2025
        ↓
[3] المسوق يشارك الرابط مع أصحاب المحلات
        ↓
[4] عندما يضغط صاحب محل الرابط:
    - يُحفظ `ref=KAMAL2025` في الـ Cookie (لمدة 30 يوم)
        ↓
[5] صاحب المحل يكمل التسجيل ويدفع الاشتراك
        ↓
[6] النظام يقرأ الـ Cookie ويربط الاشتراك بالمسوق تلقائياً
    - ينشئ سجل في affiliate_conversions
    - يحسب العمولة (مثال: 20% × 800 ج.م = 160 ج.م)
    - يُضاف للـ affiliate.balance
        ↓
[7] المسوق يرى لوحة تحكم بعملائه وعمولاته
        ↓
[8] Super Admin يراجع ويعتمد العمولات ويدفعها
```

### 6.2 لوحة تحكم المسوق (Affiliate Dashboard)

الصفحات المطلوبة:
1. **الرئيسية:** إجمالي الكسب، الرصيد المعلق، عدد العملاء النشطين.
2. **رابط التسويق:** عرض الرابط الفريد مع زر "نسخ" + QR Code.
3. **سجل العملاء:** قائمة المحلات التي سجلت عبر رابطه (الاسم، تاريخ الاشتراك، الباقة، الحالة).
4. **سجل العمولات:** كل عمولة مع حالتها (Pending / Approved / Paid).
5. **طلب سحب:** المسوق يطلب سحب رصيده (Payout Request).

### 6.3 صلاحيات المسوق

| الصلاحية | متاح للمسوق |
|---|:---:|
| رؤية لوحة التحكم الخاصة | ✅ |
| رؤية عملائه فقط (لا بيانات المحلات) | ✅ |
| رؤية عمولاته | ✅ |
| طلب سحب الرصيد | ✅ |
| تعديل أسعار العمولات | ❌ (Super Admin فقط) |
| رؤية عملاء مسوقين آخرين | ❌ |
| الدخول لأي بيانات Tenant | ❌ |

### 6.4 قواعد عمل الأفلييت (Business Rules)

| # | القاعدة |
|---|---|
| AF-01 | **العمولات المتكررة:** يحصل المسوق على 20% من أول اشتراك + 10% من كل تجديد لاحق. |
| AF-02 | إذا ألغى المحل اشتراكه قبل 30 يوم → العمولة تُلغى (Clawback). |
| AF-03 | الـ Cookie صالحة 30 يوماً — إذا سجّل بعدها لا عمولة. |
| AF-04 | إذا استخدم العميل كودين مختلفين → يُعتمد **آخر** رابط ضُغط. |
| AF-05 | الحد الأدنى للسحب: 500 ج.م. |
| AF-06 | مراجعة الـ Super Admin إلزامية قبل الدفع. |

## 8. مسارات الـ Authentication (3 مسارات منفصلة)

```
/admin/*           → Super Admin Auth  (Central DB: saas_admins)
/affiliate/*       → Affiliate Auth    (Central DB: affiliates)
/app/*             → Tenant User Auth  (Tenant DB: users + TenantMiddleware)
```

كل مسار له Guard منفصل في Laravel:
```php
// config/auth.php
'guards' => [
    'admin'     => ['driver' => 'jwt', 'provider' => 'saas_admins'],
    'affiliate' => ['driver' => 'jwt', 'provider' => 'affiliates'],
    'tenant'    => ['driver' => 'jwt', 'provider' => 'users'],
],
```
