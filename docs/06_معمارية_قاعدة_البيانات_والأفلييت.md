# 06 معمارية قاعدة البيانات ونظام الأفلييت
**مشروع Ebraa ERP - الدليل المرجعي للمطور**

---

## 1. معمارية قاعدة البيانات المزدوجة (Dual Database Architecture)
بناءً على قرار استخدام **stancl/tenancy**، سيتم تقسيم النظام إلى قاعدتي بيانات منفصلتين تماماً لضمان عزل بيانات المشتركين (Tenants) وأمانها.

### أ. قاعدة البيانات المركزية (Central Database)
تخزن البيانات العامة لمنصة الـ SaaS والتحكم في المشتركين.

| الجدول | الوصف |
|---|---|
| `tenants` | بيانات المشتركين (المحلات)، يشمل الـ ID والنطاق (Domain). |
| `domains` | النطاقات أو النطاقات الفرعية المرتبطة بكل Tenant. |
| `plans` | باقات الاشتراك (Basic, Pro, Business). |
| `subscriptions` | سجل اشتراكات المحلات وتواريخ الانتهاء. |
| `affiliates` | بيانات المسوقين (الاسم، كود الأفلييت، نسبة العمولة). |
| `affiliate_payouts` | سجل طلبات سحب الأرباح للمسوقين. |

### ب. قاعدة بيانات المحلات (Tenant Database)
يتم إنشاء نسخة من هذه الجداول لكل محل (Tenant) جديد، أو استخدام قاعدة بيانات مشتركة مع عزل `tenant_id` (حسب الإعداد المختار في stancl/tenancy). **القرار المعتمد: Separate Database per Tenant لضمان أقصى أداء وأمان.**

| الجدول | الوصف |
|---|---|
| `users` | الموظفون داخل المحل (صاحب المحل، مدير، بائع). |
| `categories` | تصنيفات المنتجات (قطع غيار، أجهزة، إكسسوارات). |
| `products` | المنتجات العامة (اسم، وصف، سعر تكلفة، سعر بيع). |
| `device_items` | **الجدول الأهم:** يحتوي على السيريالات الفرعية لكل قطعة (Serial/IMEI). |
| `sales` | فواتير المبيعات. |
| `sale_items` | تفاصيل الأصناف داخل الفاتورة. |
| `customers` | بيانات العملاء النهائيين وسجل ديونهم. |
| `maintenance_tickets` | تذاكر الصيانة (للأجهزة التي تحتاج إصلاح). |

## 2. نظام الأفلييت (Affiliate System Logic)
يعمل النظام بشكل مركزي (Centralized) لخدمة كافة الـ Tenants.

### تدفق العمل (Workflow):
1. يسجل المسوق في بوابة الأفلييت ويحصل على رابط فريد: `ebraa-erp.com?ref=KAMAL10`.
2. عندما يضغط صاحب محل على الرابط، يتم تخزين `affiliate_id` في الكوكيز.
3. عند إتمام صاحب المحل لعملية الاشتراك والدفع، يقوم النظام بـ:
    - ربط الـ `tenant_id` بـ `affiliate_id` في قاعدة البيانات المركزية.
    - حساب العمولة (مثلاً 20%) وإضافتها لرصيد المسوق.
4. يمكن للمسوق طلب سحب رصيده عند وصوله للحد الأدنى عبر لوحة تحكمه.

## 3. التكامل مع تليجرام (Telegram Notifications)
بدلاً من الواتساب، سنستخدم **Telegram Bot API** لإرسال التنبيهات.

### الوظائف:
- **للتاجر:** استقبال تنبيه عند كل عملية بيع أو جرد أو وصول صنف لحد الطلب.
- **للعميل:** استقبال فاتورة البيع بصيغة PDF أو رابط عبر تليجرام.
- **للمسوق:** تنبيه فور اشتراك محل جديد عن طريقه.

---

## 4. الفئات المستهدفة (Market Verticals)
النظام مصمم ليكون عاماً (General Purpose Retail) مع تركيز تقني على "تتبع السيريالات":
- محلات الموبايلات واللابتوبات.
- محلات البلايستيشن (تتبع الأجهزة والملحقات).
- محلات الرسيفرات والدش.
- محلات قطع الغيار (إلكترونية أو ميكانيكية).
- محلات الإكسسوارات والأجهزة المنزلية.
