# 03 السبرينت والباكلوج (Sprint Structure & Backlog)
**مشروع Ebraa ERP - إدارة التنفيذ (Agile/Scrum)**
*النسخة v1.1 - مراجعة ومحسّنة*

---

## 1. هيكل العمل (Agile Setup)
- **دورة السبرينت:** أسبوعان (2 Weeks)
- **إجمالي السبرينتس:** 8 سبرينتس (16 أسبوع)
- **أداة الإدارة المقترحة:** Jira (مجاني لفريق أقل من 10 أفراد)
- **تعريف Story Point:** 1 point = ~2 ساعة عمل فعلية

---

## 2. الملاحم البرمجية (Epics)

| رقم | اسم الـ Epic | الوصف | السبرينت |
|---|---|---|---|
| **E1** | Foundation | البنية التحتية، Auth، Multi-tenancy | S1 |
| **E2** | Inventory | المنتجات، السيريال، الجرد | S2, S3 |
| **E3** | POS & Billing | نقطة البيع، الفواتير، المرتجعات | S4, S5 |
| **E4** | CRM | العملاء، الموردون، أوامر الشراء | S4, S5 |
| **E5** | BI & Reports | التقارير، الخزينة، لوحة التحكم | S6 |
| **E6** | SaaS Admin | الاشتراكات، الدفع، إدارة المحلات | S7, S8 |

## 3. تفاصيل السبرينتس الكاملة (MVP Backlog)

### 🗓️ سبرينت 1 — البنية التحتية (Foundation)
**Sprint Goal:** تهيئة بيئة تطوير مستقرة ونظام عزل البيانات.
**إجمالي النقاط:** 29 Points

| Task ID | العنوان | الوصف | AC | Points |
|---|---|---|---|---|
| **STP-001** | إعداد Laravel 11 + Docker | تثبيت Laravel، إعداد docker-compose بـ PHP, Nginx, PostgreSQL, Redis | Docker يعمل بأمر واحد | 3 |
| **STP-002** | CI/CD Pipeline | إعداد GitHub Actions لتشغيل الـ Tests تلقائياً عند كل Push | كل push يشغل الـ Tests | 3 |
| **STP-003** | إعداد stancl/tenancy | تثبيت وإعداد الحزمة لإدارة عزل البيانات (Dual DB) آلياً | اختبار يثبت عزل البيانات بين الجداول | 8 |
| **STP-004** | مصفوفة الصلاحيات المتقدمة | Spatie Permissions مع واجهة مصفوفة CRUD لكل شاشة وأعلام ميزات (Feature Flags) | دعم CRUD لكل موديول + Flags مثل تعديل السعر | 8 |
| **STP-049** | CRUD الفروع والمخازن | إنشاء الفروع والمخازن التابعة لكل Tenant مع إعداداتها الخاصة | فرع جديد يُنشأ بنجاح | 5 |
| **STP-050** | ربط المستخدمين بالفروع | جدول `user_branches` + منطق اختيار الفرع عند الدخول | المستخدم يدخل على فرعه المخصص فقط | 5 |
| **STP-005** | Authentication API | Login, Logout, Password Reset, JWT Token | كل Endpoint مغطى باختبار | 5 |
| **STP-006** | Tenants CRUD | إنشاء محل جديد (Tenant) بإعداداته الأساسية | محل جديد يُنشأ بنجاح من Super Admin | 5 |
