# 🚀 Ebraa ERP (إبراء) — وثائق المشروع الكاملة

> **نظام SaaS متكامل يعمل كجهاز عصبي للمنشأة، يربط كافة الأقسام بدقة ونزاهة.**
> **إبراء (Ebraa ERP):** يجمع بين الدقة المالية والمعنى اللغوي العميق لـ "النزاهة والخلاص من العيوب"، مما يعكس التكامل، الوضوح، والثبات الإداري.
> تتبع كل قطعة برقمها التسلسلي (Serial/IMEI) — سحابي، سريع، وبسعر يناسب السوق المصري والخليجي.

## 💼 دراسة الجدوى للمستثمر

📄 **[ebraa_erp_investor_deck.md](./ebraa_erp_investor_deck.md)**

الملف الموجه للمستثمر أو الشريك الاستراتيجي. يحتوي على:

- **حجم السوق:** أكثر من 100,000 محل إلكترونيات في مصر والخليج.
- **مقارنة المنافسين:** جدول سريع يوضح لماذا Ebraa ERP هو الخيار الأذكى.
- **نموذج التوفير:** Solo Developer + AI = تكلفة بناء **83,000 ج.م** بدلاً من 1.6 مليون.
- **الجدوى المالية:** نقطة التعادل عند **30 عميل فقط**.
- **التسعير الديناميكي:** زيادة سنوية 15% لمواكبة التضخم.
- **توقعات الأرباح:** من 12,500 ج.م/شهر (50 عميل) حتى 856,000 ج.م/شهر (1000 عميل).
- **التوسع الخليجي:** أهداف السوق السعودي والإماراتي بالريال السعودي.
- **سيناريوهات التمويل:** 3 سيناريوهات للشراكة (مستثمر مالي / شريك استراتيجي).

## ⚙️ الدراسة التقنية

📄 **[ebraa_erp_technical_study.md](./ebraa_erp_technical_study.md)**

الدراسة التقنية الموجهة لمن يريد فهم القرارات المعمارية. يحتوي على:

- **سبب اختيار Laravel Monolith** بدلاً من Microservices (توفير 40% من وقت التطوير).
- **مقارنة تقنية** مع منافسي السوق (SaaS vs Desktop).
- **تفاصيل الـ Tech Stack:** Laravel 11 (PHP 8.2+), Vue.js 3, Inertia.js v3.0, PostgreSQL, Redis.
- **استراتيجية Multi-tenancy:** Single Database + Row Level Isolation.
- **استراتيجية التسعير** والزيادة السنوية المركبة (15%).
- **Product Roadmap** مقسّم لـ 3 مراحل (MVP → صيانة → أتمتة).
- **منظومة المنتجات:** E-commerce، HR Module، Integration Layer.

## 🔍 تحليل المنافسين

📄 **[egyptian_erp_competitor_analysis.md](./egyptian_erp_competitor_analysis.md)**

تقرير شامل عن المشهد التنافسي في السوق المصري (2025/2026). يحتوي على:

- **أنظمة السحاب (SaaS):** تحليل إدارة (Edara)، دفترة (Daftra)، وافق، فيكس ديسك — مع الأسعار والمميزات والعيوب.
- **أنظمة الديسكتوب:** إيزي ستور، ديكسيف، المنارة، كويك سوفت — وأسباب تراجعها.
- **مقارنة تتبع السيريال:** كيف يتعامل كل نظام مع IMEI Tracking.
- **حساب TCO:** تكلفة الملكية الإجمالية لمحل بـ 3 فروع على مدى 3 سنوات (SaaS vs Desktop).
- **الخلاصة التنافسية:** الثغرة التي يملؤها Ebraa ERP.

## 🏗️ docs/06 — المعمارية والأفلييت

📄 **[docs/06_معمارية_قاعدة_البيانات_والأفلييت.md](./docs/06_معمارية_قاعدة_البيانات_والأفلييت.md)**

**أهم ملف في المشروع من الناحية المعمارية.** يحتوي على:

- **التعارضات المكتشفة:** جدول بكل الأخطاء المعمارية التي تم اكتشافها وتصحيحها.
- **معمارية قاعدتي بيانات:**
  - **Central DB:** `plans`, `tenants`, `subscriptions`, `saas_admins`, `affiliates`.
  - **Tenant DB:** كافة بيانات المحلات معزولة بـ `tenant_id`.
- **SQL Schema كامل** لكل جدول في قاعدتي البيانات.
- **كود Laravel** الصحيح لـ `CentralModel`، `TenantModel`، و`TenantMiddleware`.
- **نظام الأفلييت (Affiliate System):**
  - تدفق عمل كامل: من تسجيل المسوق حتى سحب العمولة.
  - صفحات لوحة تحكم المسوق (5 صفحات مفصّلة).
  - 6 قواعد عمل (Clawback، حد أدنى، Cookie 30 يوم).
  - 6 Tasks جاهزة للإضافة لـ Sprint 7.
- **3 مسارات Auth منفصلة:** `/admin`, `/affiliate`, `/app`.
