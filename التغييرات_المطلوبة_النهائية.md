## 1. نظام الألوان — إعادة بناء كاملة

### المشكلة
الألوان الحالية لا توصل معنى — كل الحالات تبدو متشابهة بصريًا.

### التغيير المطلوب

استبدل الألوان الحالية بـ **Color Token System** ثابت:

```css
/* حالات المخزون */
--status-danger:  #E24B4A;   /* نفذ المخزون   — أحمر  */
--status-warning: #BA7517;   /* ناقص المخزن   — برتقالي */
--status-ok:      #3B6D11;   /* متوفر          — أخضر  */

/* خلفيات الحالات */
--status-danger-bg:  #FCEBEB;
--status-warning-bg: #FAEEDA;
--status-ok-bg:      #EAF3DE;
```

### شكل Badge الحالة بعد التغيير

| الحالة | الشكل المطلوب |
|---|---|
| نفذ المخزون | خلفية حمراء فاتحة + نص أحمر غامق + نقطة حمراء |
| ناقص المخزن | خلفية برتقالية فاتحة + نص برتقالي غامق + نقطة برتقالية |
| متوفر | خلفية خضراء فاتحة + نص أخضر غامق + نقطة خضراء |

> ❌ لا تستخدم نفس الحجم ونفس الرمادي للثلاث حالات
> ✅ كل حالة لازم تتعرف عليها من بعيد بدون قراءة النص

## 2. أزرار الإجراءات — تكبير وإعادة ترتيب

### المشكلة
- الأزرار صغيرة جدًا (حوالي 24px) — الأصابع مش بتضغط بدقة
- زر الحذف (أحمر) جنب زر التعديل مباشرة — خطر خطأ عرضي
- الأزرار في أقصى اليسار والمستخدم شغال من اليمين

### التغييرات المطلوبة

**الحجم:**
```
الحد الأدنى لكل زر: 48px × 48px
```

**الترتيب الجديد داخل كل صف:**

```
[ اسم المنتج + الحالة ]  |  الكمية  |  [ ✏️ تعديل ]  [ ↔️ نقل ]  [ 🗑️ حذف ]
```

- زر التعديل والنقل: بجانب بعض
- زر الحذف: مفصول بمسافة 16px على الأقل عنهم
- كل زر: أيقونة + نص قصير (مش أيقونة وحدها)

**مثال:**
```
✏️ تعديل    ↔️ نقل    |    🗑️ حذف
```

## 3. الإجراءات قريبة من المستخدم

### المشكلة
المستخدم بيقرأ اسم المنتج على اليمين، الأكشن على أقصى اليسار.

### التغيير المطلوب

**خيار 1 — الأسهل:**
انقل أزرار الإجراءات لتكون العمود الثاني بعد اسم المنتج مباشرة (على اليمين).

**خيار 2 — الأفضل للـ UX:**
اجعل الضغط على الصف نفسه يفتح **Quick Action Bar** في مكانه:

```
[ ✏️ تعديل الكمية ]  [ ↔️ نقل لمخزن ]  [ 🗑️ حذف ]  [ ✕ إغلاق ]
```

يظهر تحت الصف المضغوط مباشرة، ويختفي لما المستخدم يضغط أي حاجة تانية.

## 4. صفوف الجدول — تكبير الحجم

### المشكلة
ارتفاع الصف حوالي 40px — صعب على اللمس والقراءة.

### التغيير المطلوب

```
ارتفاع كل صف: 64px على الأقل
حجم خط اسم المنتج: 15px bold
حجم خط رقم الكمية: 18px bold
```

---

## 5. أعمدة الجدول — إعادة أولوية

### الترتيب الحالي (من اليمين)
```
اسم المنتج | كود المنتج | التصنيف | المخزون | السعر | الإجراءات
```

### الترتيب المقترح
```
الحالة | اسم المنتج | الكمية | الإجراءات السريعة
```

- كود المنتج، التصنيف، السعر: تنضهر لما المستخدم يضغط على الصف (Expandable Row)
- المستخدم البسيط محتاج يعرف 3 حاجات بس: **إيه المنتج، قد إيه عنده، وإيه اللي يعمله**

---

## 6. البطاقات الإحصائية في الأعلى

### المشكلة
البطاقات بتعرض أرقام بس مفيش أكشن عليها.

### التغيير المطلوب

كل بطاقة تبقى **قابلة للضغط** وتفلتر الجدول أوتوماتيك:

| البطاقة | عند الضغط |
|---|---|
| نواقص المخزون: 45 | يفلتر الجدول ويعرض الناقصين بس |
| إجمالي الأصناف: 153 | يعرض الكل (يمسح الفلتر) |

أضف على كل بطاقة سهم صغير أو نص "اضغط للعرض" عشان المستخدم يعرف إنها clickable.

## 7. زر "إضافة منتج"

### المشكلة
الزر موجود بس مش dominant بما يكفي.

### التغيير المطلوب
- حجم الزر: ارتفاع 48px على الأقل
- لون: أزرق قوي أو أخضر — مش بنفسجي خافت
- موقعه: يفضل في أعلى يمين الشاشة (قريب من عين المستخدم العربي)
- النص: **`+ إضافة منتج جديد`** بخط واضح مش أيقونة + بس

---

## 8. رسائل النجاح والخطأ

### المشكلة
الرسائل الحالية صغيرة ومش واضحة.

### التغيير المطلوب

**Toast Notification كبيرة في أعلى الشاشة:**

```
✅  تم حفظ التعديل بنجاح             [×]
```
```
❌  حدث خطأ، تأكد من البيانات        [×]
```

- الخضر للنجاح، الأحمر للخطأ، البرتقالي للتحذير
- تظهر 3 ثواني وتختفي أوتوماتيك
- حجم الخط: 15px على الأقل

---

## 9. تأكيد الحذف — إلزامي

### المشكلة
الحذف بيحصل مباشرة بدون تأكيد — خطر.

### التغيير المطلوب

أي ضغط على زر الحذف يفتح Dialog تأكيد:

```
┌─────────────────────────────────┐
│  هل تريد حذف "Dell Pro Book 11"؟│
│  لا يمكن التراجع عن هذا الإجراء │
│                                 │
│  [ إلغاء ]    [ نعم، احذف ]     │
│   (رمادي)        (أحمر)         │
└─────────────────────────────────┘
```

- زر الإلغاء: رمادي، على اليمين (قريب من إيد المستخدم)
- زر الحذف: أحمر، حجم أكبر
